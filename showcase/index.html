END-TO-END PIPELINE

从上传到导出 · 七步法处理链路

每一步都做了失败兜底与降级方案；长视频自动压缩切片，违规内容上传前就被拦下，最终交付清晰的文档与历史回看。

1 入口

上传 / 导入

本地单文件、批量、分片续传，或粘贴远程视频链接。文件先进入暂存区，等待安全检测。

2 安全

上传前风控

指纹黑名单 → 视觉抽帧 → 备用视觉模型 → 字幕关键词，逐层校验。命中即拒绝，不会落到正式存储区。

3 复检

分析前风控

对正式入库的视频再做一次校验，命中违规直接进入隔离目录，并把结果写入缓存避免重复扣费。

4 预处理

长视频自适应

按时长 / 体积自动压缩、必要时切片重拼。处理副本独立保存，原始视频不动，超长会自动收紧参数。

5 分析

内容分析

Whisper 转写字幕 → 模型抽取步骤；或开启视频理解直分析。失败自动降级为候选步骤或时间线摘要。

6 视觉增强

截图与看图增强

为每个步骤抽取关键截图；置信度低的步骤再次让模型"看图说话"，补强标题与描述。

7 生成

文档与字幕导出

输出 Markdown 操作指南、PDF、步骤 JSON、关键截图，同时一键导出 SRT / VTT / TXT 字幕。

✓ 交付

结果与历史

分析结果写入历史抽屉，按浏览器隔离查看。72 小时自动清理，支持单结果或批量打包下载 ZIP。

ENGINEERING DETAILS

工程化优化与落地实践

从前端渲染、网络层、上传链路、长视频处理到 LLM 调用、风控与可观测性，每一处都做了真实的取舍与回退方案。

前端渲染性能
  • 历史抽屉走自研虚拟列表：只渲染当前可见的几行，上下各预留一点缓冲，万条记录滚动也不掉帧
  • 结果区与超长 Markdown 区滚动到才渲染，离屏部分不占用渲染线程
  • 关键容器声明独立的渲染边界，让浏览器把它们放进单独的合成层，避免改动一处导致整页重排
  • 动画走 GPU 合成（位移 / 透明度），不走重排路径，主线程不卡顿
虚拟列表 按需渲染 GPU 合成
移动 / 弱设备自适应
  • 启动时自动识别移动端、低端机、省流量模式、关闭动画偏好，命中任一即切到性能模式
  • 性能模式下 Hero 动画、流光按钮、平滑滚动等装饰性效果自动降级为静态，避免低端机帧率塌陷
  • 分析进度的轮询频率自适应：桌面 5 秒 / 移动 9 秒，节流网络与电量
  • 所有动画都尊重系统的"减少动效"偏好，无障碍优先
设备识别 无障碍 省电策略
分片上传与断点续传
  • 大文件默认按 8MB 一片分块上传，单文件最大支持 500MB
  • 后端会告诉前端"已经收到了哪些片"，前端只补传缺口，已上传的部分永远不会重传
  • 上传会话记在浏览器本地，刷新页面、切网络、临时断电都能从断点继续，传完自动清理
  • 小文件走内存缓冲提速，大文件落到磁盘临时文件，避免高并发吃光内存
断点续传 8MB 分片 内存 / 磁盘混合
长视频自动预处理
  • 按视频时长 + 体积分四档：标准 / 长视频 / 超长 / 必须先裁剪
  • 长视频自动压缩、必要时切片重拼，处理副本只用一次，原始视频不动
  • 超长档自动关掉视频理解、关掉联网搜索、强制只生成摘要版，确保不超时不爆显存
  • 超过 90 分钟或 500MB 直接拒绝，并明确告诉用户"建议先裁剪"，比中途失败更友好
自动切片 参数自动收紧 早拒绝早友好
多模型平台统一接入
  • 每个模型平台显式声明自己支持什么：能聊天、能上传文件、能看视频、能联网搜索
  • 同一段调用代码，根据配置自动路由到 Ark · OpenAI · DeepSeek · Qwen，业务代码不感知差异
  • 切到不支持视频理解的平台，系统自动改走"字幕抽取"路线，并在前端如实展示当前用了哪条链路
  • 用户调用了平台不支持的能力时，返回的不是 500 报错，而是"当前平台不支持视频理解，请切换或关闭"这种可读提示
能力声明 自动选路 可读错误
四层内容风控 + 结果缓存
  • 四层逐级校验：文件指纹黑名单 → 视觉抽帧识别 → 备用视觉模型 → 字幕关键词词库，前一层不行就回退到下一层
  • 上传时与开始分析前各跑一次风控，命中违规的视频被移到隔离目录，不会污染正常分析流程
  • 对"同一个视频 + 同一份模型配置"的判定结果做缓存，重复分析不再消耗模型 API 费用
  • 风控结果统一输出"放行 / 限流 / 阻断 + 风险等级 + 命中规则码"，前后端共用同一份契约
指纹黑名单 视觉风控 省 API 费
渲染安全 & 状态持久化
  • 所有 AI 输出都先经过清洗管线再渲染，杜绝 XSS 与脚本注入，对外展示可放心信任
  • 不需要登录系统，靠每个浏览器一份匿名 ID 做多用户历史隔离，自己只看到自己的记录
  • API Key、模型选择、Whisper 等级、抽帧频率等用户偏好本地持久化，下次打开自动还原
  • 分片上传的进度也存在浏览器本地，断网或关页都能续传
输出清洗 本地偏好 无登录隔离
错误处理 & 引导式 UX
  • 统一错误层把后端的状态码、风控原因、分段策略等信息拼成一句完整可读的错误说明，避免"请求失败"这种黑盒提示
  • 用户漏填 API Key 或模型配置时，自动展开对应抽屉、聚焦输入框，并配脉冲光圈引导视线，比红字提示更直观

# 动画与性能优化记录

更新时间：2026-03-14

## 1. 抽屉打开卡顿优化

### 1.1 抽屉改为常驻挂载
- 历史抽屉和设置抽屉不再按打开状态卸载/挂载。
- 通过 `hidden` 与 `aria-hidden` 控制显隐，减少首次打开时的大量 DOM 构建开销。
- 位置：`web-react/src/App.tsx`

### 1.2 抽屉动画降载
- 将部分 keyframe 动画改为更轻量的 `transition`。
- 保留必要的进入动效，减少子元素级联动画造成的主线程压力。
- 位置：`web-react/src/index.css`

## 2. 轮询与状态更新去重

### 2.1 进度文案去重更新
- 新增 `setProgressTextIfChanged`，仅当文本变化时才触发 `setProgressText`。
- 应用于单文件轮询、批量轮询、分片上传提示。
- 位置：`web-react/src/App.tsx`

### 2.2 进度面板对象去重更新
- 新增 `isSameProgressBoard`，在 `updateProgressBoard` 中比较新旧对象。
- 无实际变化时返回旧 state，避免无意义重渲染。
- 位置：`web-react/src/App.tsx`

### 2.3 批量文件状态去重更新
- 批量轮询中仅在文件状态确实变为 `processing` 时才更新 `batchFiles`。
- 无变更时返回旧数组，降低列表重渲染频率。
- 位置：`web-react/src/App.tsx`

### 2.4 轮询频率
- 分析轮询为每 2 秒一次（`setInterval(..., 2000)`）。
- 位置：`web-react/src/App.tsx`

## 3. 重内容区域渲染优化

### 3.1 结果区组件拆分与 memo
- `ReadonlyStepsList` 与 `MarkdownPreview` 使用 `memo`。
- 抽屉开关时减少“识别步骤”和“总结文档”大块 DOM 的重复渲染。
- 位置：`web-react/src/App.tsx`

### 3.2 结果区渲染隔离
- `results-grid` 增加 `contain`。
- 结果卡片增加 `content-visibility: auto` 与 `contain-intrinsic-size`。
- 位置：`web-react/src/index.css`

## 4. 历史长列表虚拟化

### 4.1 历史记录窗口化渲染
- 新增 `VirtualizedHistoryList`，按可视区域渲染历史项。
- 核心参数：
  - `HISTORY_VIRTUAL_ITEM_HEIGHT = 74`
  - `HISTORY_VIRTUAL_OVERSCAN = 6`
- 使用 `scrollTop + viewportHeight` 计算渲染窗口，显著减少长列表节点数量。
- 位置：`web-react/src/App.tsx`

### 4.2 首开与尺寸同步
- 使用 `ResizeObserver` 同步可视高度。
- 抽屉变为激活状态时，通过 `requestAnimationFrame` 做一次高度同步，避免首帧渲染窗口不准确。
- 位置：`web-react/src/App.tsx`

### 4.3 虚拟项样式隔离
- 新增 `.history-virtual-item { contain: layout paint; }`。
- 降低单项更新对其他区域的影响。
- 位置：`web-react/src/index.css`

## 5. 背景光束组件能力优化（可选控制）

- `BackgroundBeams` 组件支持 `animated` 参数，可在需要时切换静态/动态模式。
- 渐变随机参数通过 `useMemo` 固定，避免重复渲染时反复生成随机值。
- 当前页面实际配置为常态播放（未在抽屉打开时暂停）。
- 位置：
  - `web-react/components/ui/background-beams.tsx`
  - `web-react/src/App.tsx`
